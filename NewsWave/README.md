# NewsWave

Веб-версия NewsMaker на ASP.NET Core и DevExtreme.

## Что повторено из NewsMaker

- экран «Установки» с настройками запуска, BridgeNote и почтового сервера;
- экран «Отчёт» с командами «Начать рассылку» / «Остановить» и «Отправить мне»;
- ручной и автоматический запуск по дню недели и времени;
- процедуры chk_news и chk_subscribers;
- формирование Excel из представления new_prix_list;
- адресный буфер v_ss_buffer;
- персональный текст, ссылка отписки и архив новинок;
- удаление успешно отправленной записи из subscribers_send_buffer;
- интервалы в subscribers_send_setting.

Рассылка и отправка отчёта выполняются через настроенный SMTP вашего домена.
Подписчики читаются из subscribers только по полю email. Ручное добавление выполняет
INSERT INTO subscribers (email) VALUES (...) и не придумывает значения других полей.

## Запуск

В каталоге NewsWave выполните npm install, затем dotnet run.

DevExtreme 25.2.7 устанавливается локально командой npm install.
Настройки, введённые на странице NewsMaker, сохраняются в
App_Data/newsmaker-settings.json; каталог исключён из Git.

До настройки BridgeNote и SMTP запуск рассылки завершится безопасной ошибкой,
не выполняя отправку.
## Возможности интерфейса

- обзор состояния BridgeNote, очереди, расписания и последнего запуска;
- редактор письма с библиотекой шаблонов и HTML-предпросмотром;
- тестовая отправка на DeveloperEmail перед массовым запуском;
- импорт подписчиков из CSV/TXT, поиск, удаление и подавление дублей;
- сохранение последних 100 завершённых запусков в App_Data/run-history.json.

Пароли BridgeNote и SMTP не возвращаются в HTML и сохраняются в
App_Data/newsmaker-settings.json в защищённом виде через ASP.NET Core Data Protection.

## Доступ администратора

При локальном запуске с 127.0.0.1 вход выполняется автоматически. Для удалённого
доступа обязательно задайте пароль окружением:

    $env:NewsWave__AdminPassword = "длинный-уникальный-пароль"
    dotnet run

Без пароля удалённые запросы блокируются. После настройки пароля NewsWave показывает
страницу входа; сессия администратора действует восемь часов.

## Проверка и публикация

Быстрая проверка ключевых страниц:

    ./scripts/smoke-test.ps1

Production-сборка:

    dotnet publish -c Release

Endpoint /health возвращает только состояние приложения и не раскрывает параметры
BridgeNote или SMTP. Каталоги App_Data и ключи Data Protection необходимо включать
в резервное копирование сервера.
